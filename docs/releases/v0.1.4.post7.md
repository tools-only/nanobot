# v0.1.4.post7

Release date: 2026-04-11

## Summary

This release makes the knowledge system domain-aware while preserving the layered archive-to-synthesis workflow.

## Highlights

- Low-level knowledge now carries explicit domain semantics:
  - `domain`
  - `subdomain`
  - `cross_domain`
  - `bridges`
- Xiaohongshu ingest now routes notes into domain-aware vault paths:
  - `raw/xiaohongshu/<domain>/`
  - `parsed/xiaohongshu/<domain>/`
  - `canonical/archive/<domain>/xiaohongshu/`
- Obsidian scaffold now pre-creates per-domain folders for archive and concept notes.
- Expansion jobs and fusion notes now retain source domain metadata and optional cross-domain bridge fields.
- Filesystem retrieval can now filter by knowledge domain through `KnowledgeQuery.domains`.

## User-Facing Changes

- Domain-specific notes can be queried independently from other archives.
- Existing layered notes can be promoted into synthesis without losing domain provenance.
- The knowledge vault structure is now clearer for mixed-domain collections such as papers, finance, and product notes.

## Migration Notes

- Existing vault notes remain readable.
- Existing notes can be moved under domain-aware directories and annotated with `domain` / `subdomain`.
- High-level synthesis remains explicit and is still not generated automatically on ingest.

## Verification

- Python compile checks passed for the updated knowledge modules and integration tests.
- Runtime smoke checks confirmed domain-filtered retrieval against the migrated vault.
- `pytest` was not available in the current environment, so the test file could not be executed end-to-end.
